# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект демонстрирует применение методов и инструментов инженерии ИИ: работа с данными, модели, пайплайны, сервис, эксперименты и воспроизводимость.

---

## 1. Паспорт проекта

- **Название проекта:** Прогноз направления движениия цен на акции
- **Автор:** Овсянников Владимир Иванович
- **Группа:** БСБО-04-23
- **Контакт:** @FenixOvi

- **Краткое описание:**  
Проект решает задачу прогнозирования направления движения цены акций на российском фондовом рынке. На основе исторических данных о ценах, макроэкономических индикаторов и новостного сентимента. Сервис принимает тикер акции и возвращает вероятность роста цены через 20 дней.

---

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта.
- `report.md` – отчёт по проекту.
- `self-checklist.md` – чеклист самопроверки.
- `notebooks/` – экспериментальные ноутбуки (EDA, baseline, LSTM, GRU, Feature Engineering).
- `src/` – основной код проекта:
  - `prepare_inference_data.py` – скрипт подготовки лёгкого датасета для инференса.
  - `predict.py` – класс для загрузки модели и предсказания.
  - `main.py` – FastAPI сервис.
  - `test_api.py` – тестовый скрипт для проверки API.
- `data/` – демонстрационные данные (генерируются скриптом, не хранятся в репозитории).
- `artifacts/` – сохранённые модели, scaler, PCA (не хранятся в репозитории).

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10`
- Зависимости из `requirements.txt`
- Для GPU‑ускорения при обучении: CUDA Toolkit (опционально)

### 3.2. Установка окружения

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение
python -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt
```

---

## 4. Как запустить проект

### 4.1. Подготовка данных для инференса (один раз после обучения)

```bash
cd project
source .venv/bin/activate
python src/prepare_inference_data.py
```

Скрипт создаст в папке `data/`:
- `pca_features_minimal.parquet` – для каждого тикера только последние 20 дней в PCA-пространстве.
- `valid_tickers.pkl` – список тикеров, на которых обучена модель.

### 4.2. Запуск сервиса (API)

```bash
cd project
source .venv/bin/activate
uvicorn src.main:app --reload
```

Сервис будет доступен по адресу: `http://localhost:8000`

### 4.3. Тестирование API

```bash
# В другом терминале
cd project
source .venv/bin/activate
python src/test_api.py
```

Или через `curl`:

```bash
curl -X POST http://localhost:8000/predict -H "Content-Type: application/json" -d '{"ticker":"SBER"}'
```

### 4.4. Эндпоинты

- `GET /health` – проверка работоспособности.
- `POST /predict` – принимает JSON `{"ticker": "SBER"}`, возвращает `{"binary_proba": 0.67}`.

## 5. Данные

- OHLCV – ежедневные цены российских акций (Kaggle `olegshpagin/russia-stocks-prices-ohlcv`).
- Макро – ключевая ставка, инфляция, M2, курс USD/RUB (ЦБ РФ).
- Новости – датасет `Kasymkhan/RussianFinancialNews`, сентимент через `rubert-tiny2-russian-financial-sentiment`.
- Признаки – более 50 технических индикаторов, макроэкономические показатели, новостной сентимент, временные метки.
- Предобработка – в ноутбуках `notebooks/eda_analysis.ipynb` и `notebooks/data_preparation.ipynb`.

## 8. Ограничения и дальнейшая работа

Текущие ограничения:
- Модель обучена на исторических данных только до 2023 года, не обновляется автоматически.
- Для новых тикеров, отсутствующих в обучении, сервис возвращает ошибку (модель не экстраполирует).
- Нет поддержки портфельных рекомендаций (только прогноз по одному тикеру).

Возможные улучшения:
- Добавить онлайн‑обучение на новых данных.
- Реализовать рекомендации по портфелю с учётом корреляций.
- Добавить кэширование предсказаний для частых тикеров.
