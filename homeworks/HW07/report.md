## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)

- KMeans "ломается" на данных с кластерами сложной формы (не сферическими) или разной плотности. В наших датасетах это особенно заметно в датасете 03.
- DBSCAN выигрывает, когда данные имеют шум и кластеры сложной формы, но сильно страдает от высокой размерности (датасет 04).
- Сильнее всего на результат влияло: 1) масштабирование (без него метрики были неадекватными), 2) наличие шумовых признаков, 3) высокая размерность для DBSCAN, 4) пропуски и категориальные признаки для датасета 04.

### 5.2 Устойчивость (обязательно для одного датасета)

- Проверка устойчивости: для датасета 02 запускали KMeans с 5 разными `random_state` (0, 42, 100, 500, 1000) и вычисляли Adjusted Rand Index (ARI) между всеми парами.
- Результаты: средний ARI = 0.8439, минимальный ARI = 0.7552.
- Вывод: KMeans показал достаточно устойчивые результаты, но не идеальные. Это может быть связано с перекрытием кластеров или их неоднородностью. Высокая устойчивость (средний ARI близок к 1) говорит о воспроизводимости результатов.

### 5.3 Интерпретация кластеров

- Интерпретация кластеров: для каждого кластера вычисляли средние значения признаков (для датасета 04 также учитывали категориальные). Например, в датасете 02 кластеры различались по значениям x1, x2 и z_noise.
- Выводы: Кластеры имеют различия по исходным признакам, что подтверждает их осмысленность. Однако без внешних меток (ground truth) сложно оценить содержательную интерпретацию. Визуализация PCA показала, что кластеры действительно разделяются в пространстве признаков. Хотя и не всегда логично.

## 6. Conclusion

1. Разные алгоритмы кластеризации подходят для разных типов данных: KMeans для сферических кластеров, DBSCAN для кластеров произвольной формы с шумом.
2. Метрики кластеризации (silhouette, DB, CH) помогают сравнивать алгоритмы, но не всегда отражают визуальное качество разбиения.
3. Препроцессинг (масштабирование, обработка пропусков и категориальных признаков) критически важен для успеха кластеризации.
4. Высокая размерность данных затрудняет кластеризацию, особенно для DBSCAN (проклятие размерности).
5. Устойчивость KMeans можно проверить с помощью ARI, и она может быть неидеальной даже на простых данных.
6. Визуализация (PCA, t-SNE) помогает оценить качество кластеризации, но может вводить в заблуждение из-за потери информации при уменьшении размерности.
7. Контроль доли шума в DBSCAN важен для получения осмысленных результатов.
8. Выбор количества кластеров в KMeans по silhouette score работает, но требует анализа всего диапазона возможных k.
